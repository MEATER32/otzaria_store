# חנות התוספים של אוצריא

אתר סטטי עבור GitHub Pages שמציג את תוספי אוצריא בצורה ציבורית, נקייה וברורה.

המטרה שלו:

- להציג תוספים לאוצריא בצורה נוחה ומושכת.
- לפתוח עמוד פרטים לכל תוסף.
- לאפשר התקנה ישירה מתוך אוצריא כאשר קישור ההורדה תומך בכך.

## מה יש כאן

- `index.html` - עמוד החנות הראשי
- `plugin.html` - עמוד פרטים של תוסף
- `data/plugins.json` - כל רשימת התוספים
- `assets/images/` - תמונות וצילומי מסך
- `assets/css/style.css` - העיצוב
- `assets/js/` - קבצי JavaScript פשוטים

## איך פותחים מקומית

אם רוצים רק לבדוק את האתר, הכי נוח לפתוח אותו עם שרת סטטי קטן.

דוגמה:

```bash
python3 -m http.server 8000
```

ואז לפתוח בדפדפן:

```text
http://localhost:8000
```

אפשר גם לפרסם ישירות דרך GitHub Pages.

## איך מוסיפים תוסף חדש

לא צריך לבנות מחדש את האתר, ולא צריך לכתוב HTML חדש.

פשוט עושים 3 דברים:

1. מעלים תמונה לתיקייה `assets/images/`
2. מוסיפים אובייקט חדש לקובץ `data/plugins.json`
3. מוודאים שקישור ההורדה באמת עובד

## תבנית לתוסף חדש

להדביק את האובייקט הבא בתוך `data/plugins.json`.

אם זה התוסף הראשון, הקובץ צריך להפוך מ־`[]` לרשימה שמכילה את האובייקט:

```json
[
  {
    "id": "my-plugin",
    "name": "השם של התוסף",
    "shortDescription": "תיאור קצר שיופיע בקוביה.",
    "description": "תיאור מלא שיופיע בעמוד התוסף.",
    "version": "1.0.0",
    "status": "stable",
    "author": "שם המפתח או הצוות",
    "updatedAt": "2026-04-28",
    "compatibleWith": "0.9.88 ומעלה",
    "tags": ["חיפוש", "מראה תואם לאוצריא"],
    "image": "assets/images/my-plugin.svg",
    "screenshots": [
      "assets/images/my-plugin-shot-1.svg"
    ],
    "downloadUrl": "https://your-domain.com/plugins/my-plugin.otzplugin",
    "homepage": "https://github.com/USER/REPO",
    "installInstructions": [
      "אם אוצריא מותקנת במחשב, אפשר ללחוץ בחנות על התקנה ישירה.",
      "מורידים את קובץ התוסף.",
      "פותחים את מסך התוספים בתוכנה.",
      "בוחרים התקנה מקובץ.",
      "מאשרים ומפעילים מחדש אם צריך."
    ]
  }
]
```

## כללים קטנים שחשוב לשמור

- `id` צריך להיות באנגלית, בלי רווחים. עדיף עם מקפים.
- `status` צריך להיות אחד מאלה: `stable`, `beta`, `experimental`
- `image` צריך להצביע על קובץ שקיים באמת
- `downloadUrl` צריך להיות קישור ישיר ונכון, ועדיף לקובץ `.otzplugin`
- אם אין צילומי מסך, אפשר להשאיר `screenshots` ריק: `[]`
- התגית `מראה תואם לאוצריא` מיועדת לתוספים שמשתמשים ב־Material 3 ומתאימים חזותית לממשק של אוצריא

## התקנה ישירה לאוצריא

כאשר `downloadUrl` מצביע ישירות על קובץ `.otzplugin`, החנות מייצרת אוטומטית קישור התקנה בפורמט הבא:

```text
otzaria://plugin/install?url=https%3A%2F%2Fyour-domain.com%2Fplugins%2Fmy-plugin.otzplugin
```

הקישור נוצר אוטומטית בכרטיס התוסף ובעמוד הפרטים, ולכן חשוב שה־URL יפנה ישירות לקובץ התוסף עצמו.

## תרומה פשוטה דרך GitHub

אם רוצים להוסיף תוסף חדש דרך GitHub בלי לעבוד הרבה עם קוד:

1. פותחים את הריפו ב־GitHub
2. נכנסים ל־`data/plugins.json`
3. לוחצים על Edit
4. מוסיפים אובייקט חדש לפי התבנית שלמעלה
5. מעלים את התמונות ל־`assets/images/`
6. פותחים Pull Request

## לפני שמסיימים

כדאי לבדוק:

- שהתמונה נטענת
- שכפתור ההורדה עובד
- שעמוד התוסף נפתח
- שאין פסיק מיותר ב־JSON
- ש־`id` של התוסף ייחודי

זהו. כל הרעיון כאן הוא לשמור על זרימה פשוטה, כדי שכל תוסף חדש ייכנס בקלות.
